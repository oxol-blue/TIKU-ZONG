# AI 兜底与计费设计

## 1. 调用流程

本地题库和 OCS 无有效答案后，按 AI 模型优先级调用 OpenAI 兼容接口；失败自动尝试下一个模型。成功结果保存 `question_ai`，同题后续命中缓存，不重复请求模型。

## 2. 安全与记录

AI Key 使用 AES-GCM 加密，管理列表只返回 `keyConfigured`。记录原题、题型、Prompt、解析答案、原始响应、provider、model、Token、耗时和状态。

## 3. 扣费

AI 使用独立额度，模型配置 `ai_charge_count` 决定每次成功调用消耗量。AI 成功并完成结果保存后扣费；网络错误、超时、非法响应和无答案均不扣费。普通题库命中使用普通额度。

## 4. 管理

管理员可筛选和查看 `question_ai`，并启用/停用缓存记录。停用只影响缓存命中，不改变“AI 结果无需人工审核即可进入 `question_ai`”的产品规则。
