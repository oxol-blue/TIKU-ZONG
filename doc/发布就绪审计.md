## 发布前必须完成

1. 在目标生产 MySQL 执行备份后运行 `go run ./cmd/migrate`，确认所有迁移版本已应用并记录版本号。
2. 配置 HTTPS、生产 JWT/加密密钥、Redis（如启用）、易支付网关和 AI Provider 密钥；禁止把这些值提交到 Git。
3. 提供实际易支付/码支付渠道的退款 API 文档后，接入真实渠道退款。当前退款流程已具备金额校验、幂等、部分退款、本地对账和套餐补偿，但不应把本地记录误解为渠道已退款。
4. 在宝塔目标机执行 Nginx、systemd、Prometheus 及告警接收器配置校验，完成 MySQL 隔离恢复演练。
5. 对真实题库、支付、AI 和 OCS 源完成端到端验收，并记录相似题误召回评估结果。

当前源码提交已成功推送到 `oxol-blue/TIKU-ZONG`；远端 CI 首次运行状态仍应在 GitHub Actions 中确认。
