# 开发过程记录

## 2026-07-12：第一迭代，项目基座

- 确认工作区为空脚手架，保留已有《题库调用系统详细开发计划》作为需求基线。
- 通过 Git Credential Manager 完成 Gitee Git 认证。
- 将 `koi-ui` 拉取到 `frontend/`，使用其 Vue 3、TypeScript、Vite、Pinia 和 Element Plus 基础能力。
- 创建 `backend/` Go module，使用 Gin 建立 API 服务入口。
- 增加环境变量示例、根目录忽略规则和健康检查接口：`GET /healthz`、`GET /api/v1/health`。
- 增加 MySQL 5.7 连接池、连接超时、迁移基线和数据库配置加载。
- 使用本地 `backend/.env` 连接远程开发数据库；该文件已被 `.gitignore` 忽略，凭据不会提交。
- 增加健康路由单元测试，并通过 `go test ./...`、`go vet ./...` 和真实 HTTP 健康检查。
- 前端依赖安装脚本按 pnpm 11 要求更新 `allowBuilds`，`pnpm run type:check` 和 `pnpm run build` 均通过。
- 增加 `users`、`refresh_tokens`、`user_api_keys` 三张 MySQL 5.7 认证表。
- 实现邮箱注册、登录、15 分钟 JWT、7 天刷新令牌、刷新令牌轮换和连续失败锁定。
- 实现用户 `/me` 接口和每用户唯一 API Key 的创建、查询接口，数据库仅保存 Key 哈希。
- 增加 `cmd/migrate` 并已将认证迁移幂等应用到远程 MySQL。
- 前端登录从 koi-ui 本地 JSON 改为调用真实认证接口，兼容后端 `code: 0` 响应。
- 端到端验证通过：注册、刷新、用户查询、API Key 创建和查询；前端类型检查与生产构建继续通过。
- 增加 `questions`、`question_options`、`question_answers` 三张 MySQL 5.7 题库表。
- 实现题目、选项、答案文字、题型、平台、科目、来源、采集时间和 SHA-256 去重字段。
- 实现管理员批量导入、普通用户登录后单题搜索和多选答案 `###` 返回。
- 导入时支持将 `A/C` 等选项编号转换为选项文字，最终答案不保存为字母编号。
- 增加题目标准化、答案排序和选项文字映射测试。
