## 2026-07-14：第四十迭代，持续集成门禁

- 新增 GitHub Actions CI 工作流：推送至 `main` 或向 `main` 提交 Pull Request 时，分别执行 Go 1.26 的测试/静态检查与 Node 22、pnpm 10 的前端依赖安装、类型检查和生产构建。
- 工作流使用锁文件缓存依赖；构建不依赖开发机的 `.env`、数据库、支付密钥或 AI 密钥。
- 更新测试计划与实施计划，明确 CI 是远端可用后的合并门禁，并记录当前 GitHub HTTP 500 推送故障。

## 2026-07-14：第三十九迭代，订单套餐实例补偿

- 对账范围调整为已支付和部分退款订单；全额退款订单没有恢复权益的必要，不再误报为“缺少套餐实例”。
- 新增事务化的套餐实例补偿：锁定缺少关联的订单，按原支付时间创建套餐实例并回写订单关联；重复执行不会重复发放。
- 服务进程每五分钟对账后自动执行补偿，管理员也可在订单管理页手动执行并重新查看对账结果。
- 补偿继续遵循时间、次数和时间次数套餐规则；时间套餐使用无限次数标记，时间次数套餐同时保留有效期与次数。

## 2026-07-14：第三十七迭代，OCS 安全自定义字段

- OCS 第三方题库源的请求 URL 和字符串参数统一支持 `${title}`、`${question}`、`${type}`、`${options}` 占位符。
- `data` 顶层字段支持安全 DSL：`value/template`、`replace`、`map`、`split`、`join`；POST 请求保留 JSON 数字与数组，GET 请求对结构化参数进行 JSON 编码。
- 明确拒绝 OCS 原生 `handler` JavaScript 配置，不在服务端执行任意脚本，避免题源配置获得代码执行能力。
- 管理后台补充主页、请求头、优先级、启用状态和 DSL 使用说明；新增 DSL 与 JavaScript 拒绝的单元测试。
