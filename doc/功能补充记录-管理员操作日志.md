# 功能补充记录：管理员操作日志

## 完成内容

- 新增迁移 `000013_admin_audit_logs.sql`，适配 MySQL 5.7。
- 对 `/api/v1/admin/` 下成功的 `POST`、`PUT`、`PATCH`、`DELETE` 自动记录操作人、动作、资源、请求路径、IP、HTTP 状态和时间。
- 新增 `GET /api/v1/admin/audit-logs` 分页查询接口，支持按管理员邮箱、资源或路径搜索。
- 管理后台新增“操作日志”页签，支持查询与分页。

## 安全边界

审计中间件从不读取或保存请求正文，因此不会把密码、JWT、API Key、AI 服务商密钥、易支付密钥或退款参数写入日志。仅操作成功（HTTP 2xx/3xx）的管理员写请求会入库。

## 部署动作

部署前在后端目录执行迁移，并重启 API 服务：

```powershell
go run ./cmd/migrate
go run ./cmd/server
```

迁移未执行时，既有管理员操作不会被阻断，但操作日志查询会因表不存在而失败；因此应先完成迁移再发布本版本。
