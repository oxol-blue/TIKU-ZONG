# 系统总体架构设计

## 1. 架构

```text
浏览器 / OCS 脚本 / API 客户端
          ↓ HTTPS
Vue 3 + koi-ui 前端     Go + Gin API
                              ↓
                    MySQL 5.7 / Redis（预留）
                              ↓
              OCS 第三方源 / AI OpenAI 兼容服务
```

前端位于 `frontend/`，单独构建部署；后端位于 `backend/`，通过 REST JSON 提供用户、题库、计费和管理接口。

## 2. 搜题链路

请求先完成 JWT 或 API Key 鉴权和 IP 策略检查，再依次尝试本地题库、OCS 源和 AI。成功结果才扣除相应额度并写调用日志；AI 结果写入 `question_ai`，后续优先使用缓存。

## 3. 安全边界

OCS 仅执行固定字段映射 DSL，不执行任意 JavaScript。AI Key 和支付 Key 使用 AES-GCM 保存。管理接口叠加管理员角色和可选 TOTP。API Key 只在创建响应中返回明文。

## 4. 部署单元

Nginx/宝塔负责 HTTPS 和静态文件，Go 服务监听内网端口，MySQL 独立存储；Redis 当前为预留能力，单节点内存限流可在后续替换为 Redis 分布式限流。
