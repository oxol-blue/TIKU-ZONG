# 功能补充记录：密码修改

## 完成内容

- 用户中心新增“安全设置”卡片，可校验当前密码、输入新密码并二次确认。
- 新增受登录保护的 `POST /api/v1/password/change` 接口。
- 后端校验当前密码、新密码长度（8–72）及新旧密码不能相同，使用 bcrypt 保存密码散列。
- 修改成功后撤销该用户所有刷新令牌，旧设备不能继续刷新会话；已签发的短期访问令牌会按原有 JWT 有效期自然失效。

## 请求示例

```json
{
  "currentPassword": "OldPassword123",
  "newPassword": "NewPassword123"
}
```
