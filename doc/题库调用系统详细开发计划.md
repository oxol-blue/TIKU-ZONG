1. 项目目标与总体方案
从零开发一个基于 koi-ui 的前后端分离题库调用系统，面向学生、教师等用户，预计支持 100～500 名用户。
技术方案：
前端：Vue 3、TypeScript、Vite、Pinia、Element Plus、UnoCSS，基于 koi-ui 二次开发。
后端：Go + Gin。
数据库：MySQL 5.7。
缓存与安全控制：Redis。
部署：Linux 云服务器 + 宝塔面板。
角色：管理员、普通用户。
语言：简体中文。
支付：易支付/码支付，采用收银台跳转 + 异步回调。
文档：全部保存到项目 doc 目录，使用 Markdown，并通过 Git 提交记录开发过程。
核心搜索链路：
API/在线搜索
  → API Key 或登录鉴权
  → 检查用户套餐额度
  → 查询 question_ai
  → 查询本地题库
  → 查询 OCS/第三方题库
  → 多来源答案合并
  → 本地无答案时调用 AI
  → 记录调用日志、AI 数据和用户反馈
  → 返回题目、答案、题型、AI 标记和耗时

3. 套餐、计费和支付
套餐类型
管理员可创建和上下架以下套餐：
时间套餐：有效期内不限普通调用次数。
次数套餐：普通次数用完后失效，不限制时间。
时间次数套餐：有效期或次数任一条件达到即失效。
AI 次数作为独立额度管理。
套餐支持：
自定义开始时间和结束时间。
普通次数。
AI 次数。
限购。

免费套餐。
上下架。
优惠券。
套餐叠加。
用户等级限制预留。
购买数量限制。
用户有多个套餐时：
用户可以手动选择套餐。
未指定时自动使用最早到期的可用套餐。
套餐实例独立记录，不直接合并成无法追溯的余额。
购买后生成独立套餐实例和消费流水。
扣费规则
默认采用“成功结果才扣费”：
本地题库成功返回：扣普通调用额度。
第三方/OCS 成功返回：扣普通调用额度。
AI 成功返回：扣 AI 额度，可按管理员配置额外记录 Token 或成本。
参数错误、无可用套餐、系统异常、超时、题库无结果、AI 失败：不扣费。
批量接口暂不开放，首期单题请求固定按一次调用计算。
所有扣费使用消费流水记录，避免只修改余额导致账目无法追溯。
余额扣减使用 MySQL 事务和行锁，避免并发重复扣费。

4. API 与 OCS 兼容
普通 API
首期只提供单题搜索：
GET /api/v1/search?key={api_key}&q={question}
请求参数：
key：用户 API Key。
q：纯文本题目。
可选题型、选项、平台、科目参数。
返回字段：
{
  "code": 0,
  "message": "success",
  "data": {
    "question": "题目内容",
    "answer": "答案内容",
    "type": "single",
    "is_ai": false,
    "search_time": 123,
    "sources": []
  }
}
API 规则：
首期使用 Query 参数传 Key。
开放跨域。
API Key 只显示完整值一次，数据库只保存哈希值。
API Key 支持禁用和重新生成。
首期保留限流、IP 黑白名单和防刷接口设计。
API 防刷使用 Redis 记录 IP、Key、用户维度数据。
API 失败不扣费。
任何成功答案必须写入调用日志和扣费流水。
OCS 配置生成
根据用户 API Key 自动生成 OCS 配置：
[
  {
    "name": "题库调用系统",
    "homepage": "https://your-domain.example",
    "url": "https://your-domain.example/api/ocs/search",
    "method": "get",
    "type": "GM_xmlhttpRequest",
    "contentType": "json",
    "data": {
      "key": "${api_key}",
      "q": "${title}",
      "type": "${type}",
      "options": "${options}"
    },
    "handler": "..."
  }
]
系统提供：
OCS 配置生成接口。
OCS 搜索接口。
用户 Key 自动绑定。
${title}、${type}、${options} 占位符。
GET/POST。
JSON/Text 响应适配。
请求头配置。
参数模板。
返回字段映射。
单答案和多答案格式。
未找到题目时的提示格式。
题目为空时保留 OCS 原始题目。
多选答案使用 ### 分隔。
OCS 安全兼容策略
由于系统明确不允许执行任意 JavaScript，首期不直接执行 OCS 的 handler 字符串，而采用安全解析 DSL：
字段路径读取，例如 data.answer。
数组映射。
字段重命名。
字符串替换。
字符串分割和拼接。
条件判断。
成功码判断。
单条结果和多条结果映射。
固定字段默认值。
多选答案分隔符配置。
任意 JavaScript handler 不纳入首期执行范围。后续如果确实需要完整兼容，可增加隔离进程、资源限制和无网络沙箱，但不作为当前版本默认能力。

本地 question_ai 表先查询。
如果已存在 AI 结果，不再次调用模型。
本地题库、第三方题库和 OCS 无有效答案时调用 AI。
按模型优先级依次调用。
当前模型失败、超时或返回非法数据时切换下一个模型。
所有模型失败时返回失败且不扣费。
AI 成功后写入 question_ai。
AI 成功后扣除独立 AI 额度。
保存原始题目、解析后的答案、AI 原始响应、调用时间和 Token 消耗。

11. 已确定的默认决策
用户规模：100～500 人。
不做多租户。
仅中文。
登录方式：邮箱 + 密码。
角色：管理员、用户。
邀请码：可选。
会话：短期 JWT + 刷新令牌。
API：首期单题搜索，Query 参数传 Key。
API Key：每个用户一个。
API 失败：不扣费。
成功结果：才扣费。
AI：独立 AI 额度。
AI 失败：不扣费。
AI 结果：保存到 question_ai，不重复调用。
答案合并：题库优先级 + 多数投票。
题目去重：标准化内容 + SHA-256。
相似检索：MySQL FULLTEXT/ngram。
套餐：时间、次数、时间次数三类。
套餐：允许叠加。
套餐使用：用户选择，未选择时过期优先。
支付：易支付/码支付收银台 + 异步回调。
OCS：支持配置、占位符、字段映射和安全 DSL。
OCS：首期不执行任意 JavaScript handler。
AI：DeepSeek、豆包、千问和自定义服务商。
部署：Linux 宝塔。
文档：全部保存到 doc 目录。
