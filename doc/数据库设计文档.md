# 数据库设计文档

## 1. 表分组

- 账号：`users`、`refresh_tokens`、`user_api_keys`
- 题库：`questions`、`question_options`、`question_answers`
- 计费：`packages`、`package_instances`、`package_consumptions`
- 支付：`payment_gateways`、`payment_orders`、`payment_refunds`
- AI/OCS：`ai_providers`、`ai_models`、`question_ai`、`ocs_sources`、`answerer_configs`
- 记录：`api_call_logs`、`answer_feedback`、`coupons`、`coupon_reservations`

## 2. 关键约束

题目使用 `composite_hash` 唯一约束去重；API Key 使用 `user_id` 唯一约束保证每用户一个；订单号、支付流水号和退款号唯一。题目和 AI 记录通过 `status` 逻辑停用，不物理删除。

## 3. 答案存储

`question_answers.answer_text` 保存选项对应文字，`answer_raw` 保留导入原始值；多答案拆分后按顺序保存，API 输出使用 `###` 拼接。AI 原始响应、Prompt、Token 和耗时保存在 `question_ai`。

## 4. 迁移

迁移文件位于 `backend/migrations/`，由 `go run ./cmd/migrate` 按版本顺序应用。当前已应用 `000001`～`000010`；本阶段定时任务和文档不新增迁移。
